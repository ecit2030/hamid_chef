# Discount Codes API - Postman Collection

## نظرة عامة

مجموعة Postman شاملة لاختبار API أكواد الخصم مع جميع السيناريوهات الممكنة.

## محتويات المجموعة

### 1. Authentication (المصادقة)

- **Login - Get Token**: تسجيل الدخول والحصول على Token

### 2. Validate Discount Code (التحقق من كود الخصم)

#### ✅ سيناريوهات النجاح

1. **Valid Code - Amount Above Minimum**
    - كود صالح مع مبلغ أعلى من الحد الأدنى
    - المبلغ: 200 ريال
    - الخصم المتوقع: 40 ريال (20%)

2. **Valid Code - Amount At Minimum**
    - كود صالح مع مبلغ عند الحد الأدنى بالضبط
    - المبلغ: 100 ريال
    - الخصم المتوقع: 20 ريال (20%)

3. **Valid Code - High Amount (Max Discount)**
    - اختبار تطبيق الحد الأقصى للخصم
    - المبلغ: 500 ريال
    - الخصم المتوقع: 50 ريال (الحد الأقصى)

#### ❌ سيناريوهات الفشل

1. **Invalid - Amount Below Minimum**
    - مبلغ أقل من الحد الأدنى
    - Status: 422
    - الرسالة: "الحد الأدنى للطلب هو 100 ريال"

2. **Invalid - Non-Existent Code**
    - كود غير موجود
    - Status: 422
    - الرسالة: "الكود غير موجود"

3. **Invalid - Missing Code**
    - حقل الكود مفقود
    - Status: 422
    - خطأ التحقق من الصحة

4. **Invalid - Missing Amount**
    - حقل المبلغ مفقود
    - Status: 422
    - خطأ التحقق من الصحة

5. **Invalid - Negative Amount**
    - مبلغ سالب
    - Status: 422
    - خطأ التحقق من الصحة

6. **Unauthorized - No Token**
    - الوصول بدون Token
    - Status: 401
    - الرسالة: "Unauthenticated"

## كيفية الاستخدام

### 1. استيراد المجموعة

1. افتح Postman
2. اضغط على **Import**
3. اختر ملف `discount_codes.postman_collection.json`
4. اضغط **Import**

### 2. إعداد المتغيرات

#### المتغيرات المطلوبة:

- `base_url`: عنوان الـ API (افتراضي: `http://127.0.0.1:8000`)
- `auth_token`: سيتم ملؤه تلقائياً بعد تسجيل الدخول
- `user_id`: سيتم ملؤه تلقائياً بعد تسجيل الدخول

#### طريقة التعديل:

1. اضغط على المجموعة
2. اذهب إلى **Variables**
3. عدل قيمة `base_url` إذا لزم الأمر

### 3. تسجيل الدخول

1. افتح طلب **Login - Get Token**
2. عدل بيانات تسجيل الدخول إذا لزم الأمر:
    ```json
    {
        "phone_number": "0501234567",
        "password": "password"
    }
    ```
3. اضغط **Send**
4. سيتم حفظ الـ Token تلقائياً في المتغيرات

### 4. تشغيل الاختبارات

بعد تسجيل الدخول، يمكنك تشغيل أي طلب من طلبات التحقق من الكود.

#### تشغيل طلب واحد:

1. اختر الطلب
2. اضغط **Send**
3. راجع النتيجة والاختبارات

#### تشغيل جميع الطلبات:

1. اضغط بزر الماوس الأيمن على المجموعة
2. اختر **Run collection**
3. اضغط **Run Discount Codes API**
4. راجع النتائج

## الاختبارات التلقائية

كل طلب يحتوي على اختبارات تلقائية تتحقق من:

### ✅ اختبارات النجاح

- Status code صحيح (200)
- وجود حقل `success` في الاستجابة
- وجود بيانات الخصم
- صحة حساب الخصم
- تطبيق الحد الأقصى للخصم

### ❌ اختبارات الفشل

- Status code صحيح (422 أو 401)
- وجود رسالة الخطأ
- وجود أخطاء التحقق من الصحة

## أمثلة على الاستجابات

### استجابة النجاح (200)

```json
{
    "success": true,
    "data": {
        "valid": true,
        "discount_code_id": 3,
        "code": "TEST2024",
        "type": "percentage",
        "value": "20.00",
        "original_amount": 200,
        "discount_amount": 40,
        "final_amount": 160
    },
    "message": "الكود صالح"
}
```

### استجابة الخطأ (422)

```json
{
    "success": false,
    "message": "الحد الأدنى للطلب هو 100 ريال"
}
```

### استجابة عدم المصادقة (401)

```json
{
    "message": "Unauthenticated."
}
```

### استجابة خطأ التحقق (422)

```json
{
    "message": "The code field is required. (and 1 more error)",
    "errors": {
        "code": ["The code field is required."],
        "amount": ["The amount field is required."]
    }
}
```

## استكشاف الأخطاء

### المشكلة: "Unauthenticated"

**الحل:**

1. تأكد من تسجيل الدخول أولاً
2. تحقق من أن الـ Token محفوظ في المتغيرات
3. تحقق من أن الـ Token لم ينتهِ

### المشكلة: "Connection refused"

**الحل:**

1. تأكد من تشغيل السيرفر: `php artisan serve`
2. تحقق من عنوان الـ `base_url`

### المشكلة: "The code field is required"

**الحل:**

1. تأكد من إرسال حقل `code` في الـ Body
2. تحقق من أن الـ Content-Type هو `application/json`
