# مخطط تدفق الحصول على API Token

```
┌─────────────────────────────────────────────────────────────┐
│                    تسجيل الدخول في الموقع                   │
│                         (Web Session)                        │
├─────────────────────────────────────────────────────────────┤
│  • يستخدم Cookies                                           │
│  • يعمل فقط في المتصفح                                      │
│  • لا يعمل مع API                                           │
│  ❌ لا يمكن استخدامه لإلغاء الحجز عبر API                  │
└─────────────────────────────────────────────────────────────┘

                              VS

┌─────────────────────────────────────────────────────────────┐
│                    تسجيل الدخول في الـ API                  │
│                      (Bearer Token)                          │
├─────────────────────────────────────────────────────────────┤
│  • يستخدم Bearer Token                                      │
│  • يعمل مع أي client                                        │
│  • يجب إرساله في header                                    │
│  ✅ يمكن استخدامه لإلغاء الحجز عبر API                     │
└─────────────────────────────────────────────────────────────┘
```

## الخطوات الصحيحة

```
┌──────────────────────────────────────────────────────────────┐
│ الخطوة 1: تسجيل الدخول عبر API                              │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  POST /api/login                                             │
│  {                                                           │
│    "email": "customer@example.com",                          │
│    "password": "password"                                    │
│  }                                                           │
│                                                              │
│  ↓                                                           │
│                                                              │
│  Response:                                                   │
│  {                                                           │
│    "success": true,                                          │
│    "data": {                                                 │
│      "token": "1|abc123xyz..."  ← احفظ هذا!                 │
│    }                                                         │
│  }                                                           │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│ الخطوة 2: استخدام الـ Token في طلب الإلغاء                  │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  POST /api/bookings/1/cancel-by-customer                     │
│  Headers:                                                    │
│    Authorization: Bearer 1|abc123xyz...  ← استخدم الـ Token  │
│  Body:                                                       │
│  {                                                           │
│    "cancellation_reason": "السبب هنا"                       │
│  }                                                           │
│                                                              │
│  ↓                                                           │
│                                                              │
│  Response:                                                   │
│  {                                                           │
│    "success": true,                                          │
│    "message": "تم إلغاء الحجز بنجاح"                        │
│  }                                                           │
└──────────────────────────────────────────────────────────────┘
```

## الأخطاء الشائعة

### ❌ خطأ 1: نسيان Bearer

```
Authorization: 1|abc123xyz...  ← خطأ!
```

✅ الصحيح:

```
Authorization: Bearer 1|abc123xyz...
```

### ❌ خطأ 2: استخدام session بدلاً من token

```
لا يمكنك استخدام تسجيل الدخول في الموقع!
يجب الحصول على token من /api/login
```

### ❌ خطأ 3: نسيان header

```
يجب إضافة Authorization header في كل طلب API
```

## نصائح

💡 **احفظ الـ Token** في مكان آمن (localStorage, secure storage)

💡 **استخدم نفس الـ Token** لعدة طلبات (لا تسجل دخول في كل مرة)

💡 **سجل خروج** عند الانتهاء لإلغاء الـ Token

💡 **استخدم Postman** للاختبار بسهولة
